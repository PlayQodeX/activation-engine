# activation-engine

> A Claude Code skill that loads the right rules and skills for the folder you are working in — automatically, before the first edit.

Claude's instructions are scattered: a global `~/.claude/CLAUDE.md`, a `CLAUDE.md` per repo, per-app overlays, project memory, coordination files, and skills that live globally, in plugins, and inside individual repos. Every session, Claude has to rediscover which of those actually apply here.

activation-engine indexes all of it once, then loads the exact subset that applies to your current folder — in one command. It also lets you save reusable **instances**: named bundles of skills + rules + guidelines, each with its own `/activate-<name>` command.

## Install

The skill is invoked as `/activate`, so it must live in a folder named `activate`. Clone it there, then generate its commands:

```bash
git clone https://github.com/PlayQodeX/activation-engine.git ~/.claude/skills/activate
node ~/.claude/skills/activate/instance.mjs sync   # registers the /activate-* commands
```

## Everyday commands

| Command | What it does | When to use |
| --- | --- | --- |
| `/activate` | Loads the rules, memory, coordination files and skills that apply to the current folder, then runs the pre-flight checks. | At the start of work in any repo. |
| `/activate-scan` | Rebuilds the machine index. | After install, and whenever you add/remove skills or rules. |
| `/activate-list` | Shows what the last scan found and how old it is. | To check the index is fresh (warns at 7 days). |
| `/activate-instances` | Lists your saved bundles. | To see what profiles you have. |
| `/activate-help` | This guide, and (re)writes the Markdown + HTML reference. | When you forget a command. |

## Instance commands

An instance is a saved bundle of skills + rules + guidelines for one purpose (e.g. `frontend`, `security-audit`). Each becomes its own `/activate-<name>` command.

| Command | What it does |
| --- | --- |
| `/activate create <name> --from-active` | Make a bundle from what the current folder resolves to (fastest). Captures workspace-specific skills + rules; global/plugin skills are ambient (always on) so they are omitted by default. Add them with `--include global,plugins,global-rules` (or `--include all`). |
| `/activate create <name> --skills a,b --rules p --guidelines "g1;g2"` | Make a bundle by hand. |
| `/activate add <name> --skills x` · `remove <name> --guidelines "…"` | Edit a bundle's contents. |
| `/activate-<name>` | Activate that bundle (one command per instance). |
| `/activate default <name>` then `/activate --default` | Set a per-repo default, then activate it in one flag. |
| `/activate export <name>` · `import <file>` | Move a bundle between machines (portable JSON). |
| `/activate rename <name> <new>` · `delete <name>` | Manage bundles. |

## Flags (any command)

| Flag | Meaning |
| --- | --- |
| `--json` | Machine-readable output. |
| `--list` | With `activate`: the saved-index summary. |
| `--default` | With `activate`: this repo's default instance. |
| `--path <dir>` | Scope a `create`/`activate` to a specific folder. |

## When to use what

- Fresh machine, or you added skills → `/activate-scan`
- Starting work in a repo → `/activate`
- You repeat the same setup often → make an instance, then `/activate-<name>`
- Unsure what's indexed / how old → `/activate-list`
- Forgot a command → `/activate-help`

## How activation works

Running `/activate` in a repo prints, in order: the **rule / guideline stack** (global `CLAUDE.md` + `RTK.md` → repo `CLAUDE.md` + `context.md` → app-level files → project `MEMORY.md`, sorted broad → narrow so the most specific layer wins); the **coordination / state files** (`active-tasks.md`, `blockers.md`, `known-bugs.md`); and the **skills reachable here**.

Instance skills are checked against the index and tagged **MISSING** (not on this machine), **untrusted** (a third-party plugin skill), or **new** (added since the last scan), so a bundle stays honest as skills come and go.

It degrades cleanly: a plain folder with no `CLAUDE.md`, memory or skills still produces a valid (smaller) result.

## Where things live

```text
~/.claude/skills/activate/          the skill
~/.claude/activation-engine/
  index.json                       the machine index (regenerable)
  index.md                         human-readable mirror
  instances/<slug>.json            your saved bundles
  activate-help.md / .html         this reference (generated by /activate-help)
~/.claude/commands/activate-*.md   generated slash commands
```
